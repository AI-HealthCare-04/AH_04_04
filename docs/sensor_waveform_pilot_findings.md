## 0. 요약 (한 줄)

**`decay`(보행 종료 감쇠)가 오탐을 깔끔히 가르며, 그 신호는 phase 라벨 없이 전체 스트림만
봐도(=실시간 감지기가 볼 수 있는 형태) 재현된다.** → 슬라이딩 decay 상태로 판별기 설계가 타당.
단 n=3(참여자 1명) 파일럿이라 **사람·배치 일반화는 본 수집으로 확정**해야 한다.

## 1. 데이터 품질 — 12/12 `✅ 사용 가능`

`--inspect` 전수 점검: 헤더 23컬럼 · **≈50Hz · 20.0~20.1초** · 시간축 단조 · `|a|`평균 ≈9.8(중력) ·
앉기 큐 라벨 3종 모두 `cue_delivery=success` · `sit_cue` 마커 1개/파일 · 3축 실제 변동 정상.
폐기·경고 0건. 배치는 `front_pocket_r_in / front_pocket_r_out / hand_portrait` 3종을 각 라벨이 1회씩 커버.

## 2. 오탐 재현 (§5-5)

| 라벨 / 구간 | 잘못 카운트된 걸음 (평균, n=3) |
|---|---|
| `walk_then_sit` / sitting | **1.7** (범위 0~3) |
| `sit_only` / sitting | 0.0 |

수용 기준(`walk_then_sit` 앉기 ≤ +2)에 이미 근접. (단 파일럿 n=3, 앉기 1회/trial 기준)

## 3. 판별 스코어카드 — 음성 `normal_walk/walking` vs 양성 `walk_then_sit/sitting`

| 특징 | 음성 μ | 양성 μ | Cohen d | 추천 임계 | 오분류 |
|---|---|---|---|---|---|
| vert_share | 0.422 | 0.491 | −0.37 | — | 2/6 |
| dyn_rms | 2.461 | 1.488 | 1.07 | 1.161< | 2/6 |
| fmag_std | 1.679 | 1.162 | 0.99 | 0.941< | 2/6 |
| **decay** | **1.014** | **0.141** | **14.08** | **0.557<** | **0/6** |

- **`decay`(후반/전반 동적에너지 비)가 압도적** — 정상 보행은 20초 내내 에너지 유지(≈1.0),
  `walk_then_sit`은 앉는 순간 14%로 붕괴(0.14). `#131`의 **"보행 종료 감쇠 상태"** 가설이 실측으로 맞음.
- 원래 유력 후보였던 **`vert_share`(placement 불변 수직에너지)는 약했다**(d=−0.37).
  → 판별기는 **decay 중심**으로 설계.

## 4. 실시간형(라벨 비참조) 검증 — 자명성 반박 ★

§3 의 `decay` 는 앉기 구간을 **phase 라벨로 미리 잘라** 재므로 "앉으면 멈춘다"가 부분적으로
자명하다. 실전 감지기는 라벨 경계를 모른 채 스트림을 훑어야 하므로, **phase 를 전혀 안 보고
벽시계 시간 구간만으로**(기준 1~8s vs 말미 4s, **3축 동적 벡터 RMS**) 정지 전환을 재봤다:

| label | e_ref(기준) | e_tail(말미) | 말미/기준 | 정지 전환 |
|---|---|---|---|---|
| normal_walk | 3.68 | 3.44 | 0.99 | **0/3** |
| walk_then_sit | 4.13 | 0.56 | 0.18 | **3/3** |
| sit_only | 0.31 | (기준 미달) | — | 0/3 |
| shuffle | 1.01 | 1.29 | 1.28 | 0/3 |

**`walk_then_sit`에서만 3/3 정지 전환이 잡히고 normal_walk·shuffle·sit_only 는 0.**
→ '보행 종료 감쇠'가 라벨 없이도(=실시간에도) 관측 가능. decay 접근의 **핵심 리스크(자명성)를
본 수집 전에 공짜로 제거**했다. (파라미터: `SLIDE_WIN_S/HOP_S`, `REF_*`, `TAIL_S`,
`WALK_REF_MIN_DYN`, `STOP_RATIO` — 분석 스크립트 상단 상수)

## 5. 부차 소득 — 대조군 정상 동작

- **`sit_only`**: 걸음 0 · 서있기 구간 `dyn_rms` 0.56(바닥) → 순수 앉기 기준선 정상.
- **`shuffle`**: 걸음 0~10 · `dyn_rms` 0.789(정상보행 2.46 과 정지 0.56 사이) · 정지 전환 0
  → "보행 아닌 애매한 소진폭" 대조군 역할 정상(정지 전환으로 오탐되지 않음).

## 6. 한계 (⚠️ 아직 판별기 만들 단계 아님)

## 8. 다인 확장 결과 (팀원 P01·P02 수집, 2026-07-23)

파일럿(SM-F766N) 외에 팀원 2명이 각기 다른 기기로 12 trial 씩 추가 수집(P01=SM-N986N, P02=SM-G977N).
raw 는 리포 밖 보관, 요약만 남긴다.

**① `decay` 는 여전히 압도적 1등 특징이나, 단일 고정 임계는 불완전하다(LOPO ⚠️)** — 음성
(`normal_walk/walking`) vs 양성(`walk_then_sit/sitting`), 데이터셋별로는 각각 0/6 이지만,
**P01+P02 를 풀링(#166 참여자 분할)** 하면:

| 특징 | 음성 μ | 양성 μ | Cohen d | 추천 임계 | 오분류 |
|---|---|---|---|---|---|
| dyn_rms | 1.52 | 1.09 | 0.94 | 1.359< | 3/12 |
| fmag_std | 1.16 | 0.96 | 0.59 | 1.080< | 3/12 |
| **decay** | **0.84** | **0.19** | **2.61** | **0.315<** | **1/12** |

decay 가 2등(dyn_rms d=0.94)을 크게 앞서지만, **풀링 시 오분류 0 이 아니라 1/12**. per-trial 을 뜯어보면
유일한 오버랩이 **역전**이라 어떤 단일 임계로도 못 가른다:

- 음성 최저: P01 `front_pocket_r_out` normal_walk decay = **0.349**
- 양성 최고: P02 `front_pocket_r_out` walk_then_sit 앉기 decay = **0.504**

둘 다 **`front_pocket_r_out`(헐렁한 바깥 주머니)** — 폰이 덜렁거려 정상보행 decay 를 끌어내리고(0.35),
앉은 뒤에도 잔진동이 남아 decay 가 안 떨어진다(0.50). **사람 차가 아니라 느슨한 배치의 노이즈**가 원인.
→ 판별기는 **절대 고정 임계가 아니라 그 세션 자신의 보행 기준선 대비 상대/적응 임계**로 설계해야 하며,
이는 아래 ②(실시간형)의 실패 원인과 같은 뿌리다.

**② 실시간형(라벨 비참조 슬라이딩) 파라미터는 아직 이식 안 됨 ❌** — `walk_then_sit` 정지 전환 탐지가
파일럿 3/3 → P01 1/3 → P02 0/3. 파일럿 기기에 맞춘 `STOP_RATIO=0.4`·`TAIL_S=4`·`WALK_REF_MIN_DYN=1.0`
이 다른 기기에서 미달(P02 는 앉은 뒤 말미 RMS 가 기준의 66%까지만 하강). 오검(normal/shuffle)은 0 유지.
**→ #131 감지기 PR 의 선결 과제 = 실시간 정지판정 파라미터 재설계**(절대 문턱 대신 세그먼트 decay 처럼
상대비/적응 임계 검토).

**③ `hand_portrait`(손에 들고 걷기) 는 감지기가 걸음을 0 으로 센다 — 별개 과소계수(#89)** — 원시 3축은
실제로 흔들리는데(std 1.2~1.4) 감지기 state 가 전 구간 IDLE → count 0. 손에 들면 진동 폭이 주머니의
약 1/3~1/4 라 `WalkingStepDetector` 피크 임계를 못 넘음. **#131(오탐)엔 무해**(손 배치는 앉든 걷든 0)이나
**#89 걷기 게이트엔 '손 배치 과소계수'로 기록** 필요. 이 데이터로 오탐 분석 시 `hand_portrait` 는 기여 0.
(→ 신규 이슈 #176 으로 기록.)

## 9. 실시간 정지판정 재설계 (§8-② 후속, 합본 24 trial 검증)

§8-② 의 "실시간 파라미터 미이식(P02 0/3)" 을 P01+P02 합본(각 12, 총 24 trial)으로 재설계했다.
**핵심 변경 = 절대 문턱 → 각 trial 자기 보행 기준선 대비 상대비 + 앉기 임팩트 회피.**

각 trial 을 `b`(자기 보행 기준선 = 기준구간 동적RMS 중앙값)로 정규화한 **말미/기준 비**로 표현하고,
말미 창 길이 후보 3안을 **LOPO(참여자 교차검증)** 로 비교:

| 규칙 | 말미 정의 | 정지탐지(TP/6) | LOPO 총오류 |
|---|---|---|---|
| **r4 (현행)** | 마지막 **4s** 중앙값 / b | 1/6 | 6 |
| **r2 (신규)** | 마지막 **2s** 중앙값 / b | **5/6** | **3** |
| rq | 말미 6s 중 최저 1.5s창 / b | 5/6 | 4 |

**`r2` 채택.** 현행 4s 창은 **앉는 순간 임팩트 스파이크 + 정착 과정**을 같이 물어 중앙값이 떠서
1/6 밖에 못 잡았다. 완전히 정착하는 **마지막 2s** 만 보면 5/6.

**감지기 지표로 재해석하면 사실상 무결점:**
- **과소계수 위험**(정상보행을 정지로 오판) = `normal_walk` 이 임계를 넘는 경우 → **r2 에서 normal_walk 최저 0.51 > 임계(~0.4) → 0 건.** 진짜 보행 카운트를 끊을 위험 없음.
- **과다계수 방지**(walk_then_sit 정지 포착) = **5/6.**
- LOPO 표의 "FP" 2 건 = `shuffle`-정지 · 앉기전 접근보행이 섞인 `sit_only` → 둘 다 걸음을 멈추는 게 맞는 상황이라 #131 지표엔 무해.
- 유일한 놓침(FN 1) = P02 `front_pocket_r_out` walk_then_sit `r2=0.62` — §8-① 의 그 헐렁한 주머니 잔진동 trial(앉아도 폰이 계속 흔들림, 물리적으로 애매).

**확정 스펙 (감지기 PR 입력):**
1. 정착 말미 = **마지막 2초**(4초 아님).
2. 적응 기준선 **B** = 보행 확정 구간의 이동중앙값(고정 REF창/절대바닥 대신 세션이 스스로 유지).
3. 정지 판정 = `말미2s 에너지 < k·B` (k≈0.4), `was_walking(B ≥ 보행수준)` 일 때만. 정지 시 걸음 카운트 동결(재보행 확정까지).

실시간 구현: 슬라이딩 1s 창(hop 0.5s) 3축 동적벡터 RMS → 보행 확정 시 B 갱신 → 말미 2s 가 `0.4·B` 미만이면 정지 상태 진입.
(진단 스크립트: 리포 밖 `diag_adaptive.py`, `analyze_waveform.py` 파싱·트레이스 재사용. n=24·참여자 2 라 확정 아닌 **설계 방향**; 본 수집으로 k·창길이 최종 튜닝.)
