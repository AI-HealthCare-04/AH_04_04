# Aigo 만보기 정확도 — 2차 측정 가이드 (정인님용)

**목적:** 1차 결과를 바탕으로 걸음수 정확도를 데이터로 확정하기 위한 2차 측정입니다 (이슈 #89 / A-4a). 지난번 값은 가설값이고, 이번 측정으로 최소 간격·피크 임계값을 정합니다.

> ⭐ **2차 보강(#89 완료조건 반영):** 아래 **휴대 위치(측정 5)**, **폰 흔들기(측정 4)**, **테스트 기기 기록**, **합격 기준**이 추가됐습니다. 나머지는 기존과 동일합니다.

---

## 이번에 1차와 달라진 점 3가지

1. **돌리는 다이얼이 바뀜.** 1차엔 "보행 진입 기준(8/10/12)"을 돌렸는데, 이건 카운트 정확도엔 거의 영향이 없었어요(코드 확인). 이번엔 **최소 간격(ms) × 피크 임계값(m/s²)** 을 돌립니다.
2. **시간을 같이 기록.** 화면에 "경과 초 · 케이던스(보/분) · 마지막 피크 간격(ms)"이 새로 떠요. 함께 적어주세요.
3. **타깃 = 독립 보행 시니어.** 지팡이·보행기 없이 스스로 걷는 속도(대략 60~120보/분)가 대상입니다. 보행기·지팡이급 초저속은 이번 정확도 목표가 아니에요(참고 측정만).

## 0. 준비

- **PR #113 빌드 앱 설치**(재란님이 APK 또는 브랜치 안내). 백엔드·로그인 불필요(오프라인 OK).
- 앱 → 둘러보기(데모) → 하단 **센서 탭 → 걷기 만보기**.
- 조용한 실내 6~10m 직선 공간 + 걸음을 직접 소리내어 셀 준비.
- 걸을 땐 지팡이·보행기 없이 **스스로 걷는 시니어 속도**로. (아주 느린 셔플 보행은 "측정 4 참고"에서만)
- ⭐ **테스트 기기 기록**(측정 시작 전 1회): 아래를 기록표 맨 위에 적어주세요.
  - 기기 모델명: _______________  · 안드로이드 버전: _______  · (여러 기기면 각각 표를 분리)

## 1. 화면에서 볼 것 (새로 추가된 표시 ⭐)

- **큰 숫자** = 감지된 걸음 수.
- ⭐ **"경과 N.N초 · 평균 M보/분"** (걸음 수 바로 아래) = 첫 걸음~마지막 걸음까지 시간과 평균 케이던스. → "몇 초에 몇 보" 기록용.
- **리셋** = 카운트·시간만 0으로 (튜닝 값은 유지).
- **[디버그] 마지막 피크 간격** = 방금 센 두 걸음 사이 간격(ms). ⭐ 걸음 안 이중봉우리(~250ms) vs 진짜 걸음(500ms+) 구분용 — 값이 250 근처로 자주 뜨면 한 걸음을 두 번 세는 신호.
- 🔧 **튜닝 패널** = 재빌드 없이 즉석 조절. 이번에 주로 쓰는 건 **최소 간격**과 **피크 임계값**.

## 2. 용어

- **실제 걸음** = 직접 센 진짜 발걸음. / **미탐(FN)** = 카운트 < 실제. / **오탐(FP)** = 안 걸었는데 세짐.
- 이번 핵심 = 정상보행에서 카운트가 실제(20)에 얼마나 가까운지 + 느린(독립) 보행을 안 놓치는지.

### ⭐ 합격 기준 (이 숫자에 들면 성공 — #89 목표치)

| 시나리오 | 지표 | 합격선 |
|---|---|---|
| 정상 독립보행 20보 | 카운트 | **18~22** (±10%) |
| 느린 독립보행 20보 | 미탐 | 카운트 **≥ 17** (덜 세는 건 OK, 너무 놓치면 X) |
| 앉았다 일어나기 5회 | 오탐 | **카운트 0** |
| 폰 흔들기 20회 | 오탐 | **카운트 0** |
| (참고) 초저속 셔플 | — | 목표 아님, **과다카운트만 없으면 OK** |

## 3. 측정 계획 (순서대로)

각 측정 전 **반드시 리셋**하고, 걸음은 **직접 세서 비교**하세요. 각 칸 **3회 반복**(노이즈 판별). **휴대 위치는 손(기준)** 으로 통일하고, 위치 영향은 측정 5에서 따로 봅니다.

### 측정 1 — 과다카운트 잡기 (최소 간격 스위프) ★가장 중요

피크 임계값은 **10.5 고정**. 최소 간격을 **250 → 300 → 350 → 400ms** 로 올려가며:

- **정상 20보 (3회)** → 카운트가 20에 수렴하는 최소 간격을 찾기
- **느린(독립) 20보 (3회)** → 그 값에서 느린 걸음이 안 깎이는지(케이던스로 확인)

> 기대: 최소 간격을 올리면 걸음 안 두 번째 봉우리가 병합돼 정상 20보 카운트가 34 → 20 쪽으로 내려옵니다. "마지막 피크 간격"이 250 근처였다가 500+ 로 정리되면 성공.

### 측정 2 — 느린보행 확인 (피크 임계값 스위프)

측정 1에서 고른 **최소 간격 고정**. 피크 임계값을 **10.5 → 11.0 → 11.5** 로:

- **느린(독립) 20보 (3회)** → 임계값을 올려도 안 놓치는 상한 찾기
- **정상 20보 (3회)** → 과다카운트가 더 줄어드는지

### 측정 3 — 짧은 걷기 진입기준 (선택)

보행 진입 기준을 **10 → 5** 로. 딱 **8보만** 걷기:

- 진입기준 10에선 0카운트(10 미달)일 수 있음 → 5에선 짧은 걷기가 살아나는지 확인.

### 측정 4 — 오탐 & 참고 (각 1회)

- **앉았다 일어나기 5회** → 카운트 **0 유지**가 정상.
- ⭐ **앉아서 폰 흔들기 20회**(손목 스냅으로 흔들기) → 카운트 **0 유지**가 정상. *(게임의 흔들기 기능과는 별개 — 여기선 "안 걸었는데 세지나" FP 확인만)*
- (참고) 보행기·지팡이급 아주 느린 **셔플 20보** → 정확도 목표 아님. 과다카운트만 안 나면 OK(덜 세는 건 괜찮음).

### ⭐ 측정 5 — 휴대 위치 영향 (#89 완료조건: 손·주머니·가방)

측정 1·2에서 **확정한 최소간격·피크임계 고정**. **정상 20보(각 3회)** 를 위치만 바꿔가며:

- **손**(측정 1과 같은 기준값 재확인) → **바지 주머니** → **가방(손가방/백팩)**
- 볼 것: 위치가 바뀌어도 카운트가 합격선(18~22)에 드는지, 특정 위치에서 급격히 과다/미탐 나는지.

> 흔들기는 이번 범위 밖입니다(게임 기능으로 분리). 측정 4의 흔들기는 **FP 확인용**입니다.

## 5. 팁

- 리셋은 **카운트·시간만 초기화**(튜닝값 유지) → 값 하나로 여러 번 재보기 좋아요.
- 리셋 직후 바로 걷기 시작하면 "경과 초"가 실제 걷기 시간과 잘 맞아요.
- "평균 보/분"이 **150을 넘지 않는지** 봐주세요 — 넘으면 걷기가 너무 빨라 최소 간격에 잘릴 수 있어요(시니어 속도로).
- "마지막 피크 간격"을 곁눈질로 보면 왜 과다/미탐인지 바로 감이 와요(**250 근처 반복 = 이중 카운트**).
