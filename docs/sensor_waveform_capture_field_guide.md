# 원시 파형 실기기 수집 가이드 (이슈 #131)

> **목적:** `WaveformCaptureScreen`(debug 전용)으로 '보행 직후 앉기' 분리용 라벨링 파형을
> **실기기에서 실수 없이** 120 trial 수집하기 위한 **현장 체크리스트**.
> 라벨·배치·횟수·폐기의 *규격과 근거*는 [`sensor_waveform_capture_protocol.md`](sensor_waveform_capture_protocol.md)에
> 있고, 이 문서는 그 규격을 **손에 폰을 들고 그대로 따라 하는 실전 절차**로 옮긴 것이다.
> 수집이 끝나면 [`scripts/analyze_waveform.py`](../scripts/analyze_waveform.py)(프로토콜 §5-1)로 1차 특징을 본다.

- **이슈:** #131 [A-4a #89 후속]  ·  **기기:** Samsung **SM-F766N**(폴더블)
- **상태:** 🟡 실기기 수집 대기 — 이 가이드대로 수집 → 분석 → 판별기 PR 이 #131 을 닫음

## 0. 준비물 (수집 전 1회)

| 항목 | 내용 |
|---|---|
| 빌드 | **debug APK** 설치 (`./gradlew installDebug` 또는 Android Studio Run). 릴리스엔 이 화면이 없다. |
| 기기 | **SM-F766N** — 배치 프리셋의 `fold_state`(folded/unfolded)가 이 기기 기준. |
| 음량 | **미디어 음량 > 0** 필수. 앉기 큐 비프가 안 들리면 그 trial 은 자동 폐기된다(§6). |
| 장소 | 10~15보 직선 왕복이 되는 평지. |
| 기록지 | **참여자 로그 시트**(§5) — **개인정보 금지**, `P01`·`P02` 익명 코드만. |

## 1. 앱 진입

**설정 화면 → 「🔧 파형 수집 (디버그 · #131)」** 버튼. (DEBUG 빌드에서만 노출)

## 2. 화면 구성 (버튼별)

- **라벨 선택**: `정상 보행` / `보행 직후 앉기` / `서서 앉기만` / `제자리 발끌기`
- **배치 선택**: `앞주머니(안)` / `앞주머니(밖)` / `손(세로)` → 아래에 배치 안내 문구가 뜬다.
- **🔊 비프 테스트**: 폰을 주머니에 넣기 **전에** 눌러 비프가 들리는지 확인(음량 0이면 경고).
- **● 녹화 시작**: 누른 뒤 **화면을 건드리지 않는다**(자동 진행·자동 종료).
- **■ 중단(정지)**: 비상용 — 누른 파일은 **폐기 대상**(화면 조작이 수집 구간에 섞임).
- **저장 & 공유 (CSV)**: 종료 후 눌러 CSV 를 내보내고 공유한다.

## 3. 1 trial 실행 (타이밍 고정 — 활동 15초 + 앉기 5초)

### 앉기 큐 **있는** 라벨 (`보행 직후 앉기`·`서서 앉기만`)
```
● 녹화 시작 → [15초 활동] → 🔔 비프! → 그때 앉기 → [5초] → 자동 종료 → 저장&공유
```
- `보행 직후 앉기`: 15초 **계속 걷다가** 비프가 나면 **즉시** 앉는다. (오탐 대상, §5-5)
- `서서 앉기만`: 15초 **그대로 서 있다가** 비프가 나면 앉는다. (보행 momentum 없는 순수 앉기 기준선)

### 앉기 큐 **없는** 라벨 (`정상 보행`·`제자리 발끌기`)
```
● 녹화 시작 → [20초 계속 동작] → 자동 종료 → 저장&공유
```

> 화면에 `앉기 신호까지 N초` → `🔔 앉기 신호! 지금 앉으세요` 카운트다운이 뜨지만, **비프 소리가 진짜 신호**다.
> 주머니 배치여도 소리만 듣고 앉으면 되고, 화면을 볼 필요가 없다.

⚠️ **걷기는 실제 고령자 속도(느린 편)로.** §5-5 문제 자체가 느린 케이던스(≈68 보/분)에서 생긴다 —
빠르게 걸으면 재현이 안 된다.

## 4. 수집 매트릭스 (라벨 4 × 배치 3 × 10회 = **120 trial**)

**배치 하나를 고정하고 → 4개 라벨을 각 10회** 도는 순서로 묶으면 폰을 자주 옮기지 않아 효율적이다.

| 배치 | 정상 보행 | 보행 직후 앉기 | 서서 앉기만 | 제자리 발끌기 |
|---|---|---|---|---|
| 앞주머니(안) | 10 | 10 | 10 | 10 |
| 앞주머니(밖) | 10 | 10 | 10 | 10 |
| 손(세로) | 10 | 10 | 10 | 10 |

## 5. 참여자 로그 (⚠️ 필수 — #166 데이터 누수 방지)

CSV 스키마엔 참여자 컬럼이 없다(프로토콜 부록 A). 그래서 **별도 시트**에 매 trial 을 기록한다.
저장 시 뜨는 **파일명이 그대로 `trial_id`**다.

```
trial_id,participant_id,session_id
walk_then_sit_20260724_101530_842,P01,P01_s1
normal_walk_20260724_101612_310,P01,P01_s1
...
```
- **참여자 여러 명** 확보 — 1~2명이면 참여자 단위 분할(leave-one-participant-out)이 무의미하다.
- **session_id**: 같은 사람이라도 다른 날/다른 착복 상태면 세션을 나눈다(`P01_s1`, `P01_s2`).
- 저장 위치: `Android/data/com.aihealthcare.ah0404/files/waveforms/` 의
  `<label>_<yyyyMMdd_HHmmss_SSS>.csv`.

## 6. 폐기 (수집 즉시 — 오염 데이터 배제)

아래는 바로 지우거나 별도 폴더로 격리한다(분석에 넣지 않는다):
- **■ 중단**을 눌러 끝낸 파일 — 화면 조작이 섞였다.
- **비프 전달 실패** Toast 가 뜬 파일 — `cue_delivery=pending`(비프가 안 들렸을 수 있음).
- 도중 **다른 앱 전환**으로 "백그라운드 자동 정지"가 뜬 파일 — 공백이 섞였다.

> 큐 직후 짧은 정착 구간(±300ms)은 파일 안에서 `excluded=1` 로 표시되므로 **폐기가 아니라
> 분석에서 자동 제외**된다(파일은 유지). — 스크립트가 알아서 처리한다.

## 7. 수집 후 분석

CSV 들을 한 폴더(예: `~/waveforms_raw/`)로 모으고, §5 참여자 로그를 `participants.csv`로 둔 뒤:

```bash
python scripts/analyze_waveform.py ~/waveforms_raw/ --participants participants.csv
```

→ 폐기 규칙 자동 적용 · 오탐 재현(`walk_then_sit` 앉기 구간 걸음, 현재 +8) 확인 ·
판별 스코어카드에서 유망 특징(`vert_share` 등) 지목. 자세한 출력 해석은 프로토콜 §5-1 참조.
